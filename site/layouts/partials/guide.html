Columns Explained

  • SSO: Single Sign-On support and which tier it's available on.
  • SSO Protocols: Which authentication protocols are supported (SAML, OIDC, OAuth2).
  • SCIM: Automated user provisioning support level.
  • MFA: Multi-factor authentication availability and how it can be enforced.
  • MFA Types: Which MFA methods are supported (TOTP, SMS, WebAuthn, Push, Email).
  • Compliance: Security certifications (SOC2 and ISO27001).

Tier Meanings

The tier indicates the MINIMUM plan level where the feature is available.

  • Free Available on free tier.
  • Paid Mid-tier paid plans.
  • Enterprise Highest tier only.

SCIM Support Levels

  • ✓ Full: Complete SCIM 2.0 implementation.
  • ~ Partial: Limited/Read-only features.
  • ✗ None: No SCIM support.

Symbols Used

  • ✓ Feature is available
  • ✗ Feature is not supported
  • ~ Partial or limited support
  • — Not applicable

MFA Enforcement

  • Admin: Admins can require MFA for users.
  • Optional: Users can enable, but not required.
  • Required: Mandatory for all users.
